Домашнее задание: Автоматизация E2E сценария покупки с использование паттерна Page Object Model  
Цель: Закрепить знания по паттерну Page Object Model (POM), создав и использовав его для автоматизации полного цикла покупки на сайте https://www.saucedemo.com/.
Описание: Вам предстоит автоматизировать сквозной (E2E) сценарий, который эмулирует действия пользователя на сайте: от входа в систему до завершения покупки. Вся структура вашего проекта должна быть построена на основе паттерна Page Object Model.
________________________________________
Шаг 1: Анализ сценария и создание Page Objects
Прежде чем писать код, изучите сайт и определите ключевые страницы, с которыми будет взаимодействовать пользователь. Для каждой такой страницы необходимо создать свой класс Page Object.
Обязательные Page Objects:
1.	LoginPage: Страница входа (https://www.saucedemo.com/).
○	Элементы: поле для ввода имени пользователя, поле для ввода пароля, кнопка входа, контейнер для сообщений об ошибках.
○	Методы: open(), login(username, password).
2.	InventoryPage: Страница со списком товаров (главная после логина).
○	Элементы: заголовок страницы, иконка корзины, список товаров, кнопки "Add to cart" для каждого товара.
○	Методы: addItemToCart(itemName), openCart(), getPageTitle().
3.	CartPage: Страница корзины.
○	Элементы: список добавленных товаров, кнопка "Checkout", кнопка "Continue Shopping".
○	Методы: goToCheckout() и иные на ваше усмотрение.
4.	CheckoutStepOnePage: Страница оформления заказа (шаг 1: ввод данных).
○	Элементы: поле для имени, поле для фамилии, поле для почтового индекса, кнопка "Continue".
○	Методы: fillUserInfo(firstName, lastName, postalCode).
5.	CheckoutStepTwoPage: Страница подтверждения заказа (шаг 2: обзор).
○	Элементы: информация о заказе, итоговая сумма, кнопка "Finish".
○	Методы: finishCheckout().
6.	CheckoutCompletePage: Страница с сообщением об успешном заказе.
○	Элементы: заголовок "Thank you for your order!", кнопка "Back Home".
○	Методы: getCompletionMessage().
________________________________________
Шаг 2: Написание E2E теста
Создайте новый тестовый файл (например, e2e.spec.js) и, используя созданные вами Page Objects, напишите тест, который выполняет следующий сценарий:
Сценарий для автоматизации:
1.	Открыть страницу логина.
2.	Залогиниться, используя валидные данные (standard_user, secret_sauce).
3.	Убедиться, что после логина открылась страница с товарами (проверить заголовок "Products").
4.	Добавить в корзину самый дорогой товар на странице. Подсказка: отсортируйте товар от самого дорого к дешевому и выберите первый товар.
5.	Перейти в корзину.
6.	Проверить, что в корзине находится именно тот товар, который вы добавили.
7.	Начать оформление заказа (нажать "Checkout").
8.	Заполнить информацию о пользователе (используйте любые данные, например: Имя - "Test", Фамилия - "User", Индекс - "12345").
9.	Продолжить оформление заказа (нажать "Continue").
10.	Завершить покупку (нажать "Finish").
11.	Убедиться, что заказ успешно оформлен (проверить наличие текста "Thank you for your order!").
________________________________________
Критерии оценки:
●	Структура проекта: Код должен быть организован с использованием Page Object Model. Каждый класс Page Object находится в своем файле.
●	Читаемость кода: Понятные имена переменных и методов.
●	Корректность работы теста: Тест проходит без ошибок и точно следует описанному сценарию.
●	Использование проверок (Assertions): В тесте должны присутствовать проверки ключевых моментов (например, заголовок страницы, текст сообщения об успехе, товар в корзине).
